04

Reglas de Validación

🧬 Edad — restricción biológica
Validador mode='after' — Pydantic ya convirtió el valor a int antes de validar. 0 ≤ edad ≤ 120
🏘️ Estrato — contexto colombiano
Validador mode='after' — Solo acepta enteros del sistema de estratificación socioeconómica de Colombia. 1 · 2 · 3 · 4 · 5 · 6
🗺️ Departamento — 33 entidades válidas
Validador mode='before' — Opera sobre el string crudo, normaliza capitalización y verifica contra lista oficial. Antioquia, Bogotá D.C., Valle del Cauca…
📊 Escala Likert / Porcentaje
@model_validator — Valida el rango de la respuesta según el tipo de pregunta declarado. Likert: 1–5 · Porcentaje: 0.0–100.0

Ejemplo: qué ocurre cuando se envían datos inválidos vs. válidos

❌ Payload inválido
JSON — Request
{ "encuestado": { "nombre": "Ana", "edad": 150, // ❌ > 120 "estrato": 9, // ❌ > 6 "departamento": "Texas" } // ❌ no existe }
✅ Respuesta 422
JSON — Response 422
{ "error": "Error de validación", "total_errores": 3, "errores": [ { "campo": "edad", "mensaje": "entre 0 y 120" }, ... ] }
05

Cómo usar la API

Ejemplo completo de registro de encuesta usando Python + requests

Python — POST /encuestas/
import requests payload = { "encuestado": { "nombre": "Laura Gómez", "edad": 28, "estrato": 3, "departamento": "Antioquia", "genero": "Femenino" }, "respuestas": [{ "pregunta_id": 1, "pregunta_texto": "¿Satisfacción con servicios?", "respuesta": 4, "tipo": "likert" }] } r = requests.post( "http://127.0.0.1:8000/encuestas/", json=payload ) print(r.status_code) # 201 print(r.json()) # {"id": "abc123-...", ...}

¿Por qué async def en FastAPI?

⛔ def — Síncrono (WSGI)
Un hilo por petición. Mientras espera I/O, el hilo queda bloqueado.
100 peticiones simultáneas = 100 hilos. Costoso en memoria.
Modelo de Flask / Django clásico.
✅ async def — Asíncrono (ASGI)
Un hilo cede el control con await mientras espera I/O.
Miles de peticiones concurrentes con pocos hilos. Eficiente.
Modelo de FastAPI / Starlette sobre Uvicorn.

07

Guía paso a paso

Si nunca has usado una API antes, no te preocupes. Piensa en la API como un mesero de restaurante: tú haces un pedido (petición HTTP), el mesero lo lleva a la cocina (servidor), y te trae el resultado (respuesta JSON). Aquí te mostramos exactamente cómo hacer ese pedido.

PASO 1
Abre el Swagger UI

Swagger UI es la interfaz visual de la API — como un panel de control donde puedes probar cada endpoint sin escribir código. Abre este enlace en tu navegador:

🔗 https://encuesta-api-production-9316.up.railway.app/docs

✅ Qué deberías ver: Una página con el título "API de Gestión de Encuestas Poblacionales" y una lista de endpoints con colores (verde = POST, azul = GET, naranja = PUT, rojo = DELETE).

PASO 2
Registra tu primera encuesta (POST)

El método POST sirve para crear datos nuevos. Sigue estos pasos en Swagger:

  1. Haz clic en el bloque verde POST /encuestas/
  2. Haz clic en el botón "Try it out" (aparece a la derecha)
  3. En el campo "Request body" ya hay un ejemplo cargado — puedes usarlo tal cual o modificarlo
  4. Haz clic en el botón azul "Execute"

✅ Qué deberías ver: En "Server response", un código 201 y un JSON con el campo "id" — ese es el identificador único de la encuesta que acabas de crear. Cópialo, lo vas a necesitar en los siguientes pasos.

❌ Si ves código 422: Algún dato no pasó la validación. Mira el campo "errores" en la respuesta — te dice exactamente qué campo está mal y por qué.

PASO 3
Consulta la lista de encuestas (GET)

El método GET sirve para leer datos sin modificarlos. Es como consultar sin alterar nada.

  1. Haz clic en el bloque azul GET /encuestas/
  2. Haz clic en "Try it out" → "Execute"

✅ Qué deberías ver: Un JSON con "total": 1 y la lista de encuestas registradas. Cada vez que hagas un POST, el total aumenta.

PASO 4
Busca una encuesta por ID (GET /{id})

Aquí usas el ID que copiaste en el Paso 2.

  1. Haz clic en GET /encuestas/{encuesta_id}
  2. Haz clic en "Try it out"
  3. En el campo "encuesta_id" pega el ID que copiaste
  4. Haz clic en "Execute"

✅ Código 200: Encontró la encuesta y te la devuelve completa.
❌ Código 404: No encontró ninguna encuesta con ese ID. Verifica que copiaste bien el ID.

PASO 5
Ver las estadísticas (GET /estadisticas/)

Este endpoint es especial — calcula estadísticas en tiempo real sobre todas las encuestas registradas.

  1. Haz clic en GET /encuestas/estadisticas/
  2. Haz clic en "Try it out" → "Execute"

✅ Qué deberías ver: Promedio de edad, edad mínima y máxima, distribución por estrato y por departamento de todas las encuestas en el sistema.

PASO 6
Elimina una encuesta (DELETE)

El método DELETE elimina permanentemente un registro.

  1. Haz clic en DELETE /encuestas/{encuesta_id}
  2. Haz clic en "Try it out"
  3. Pega el ID de la encuesta a eliminar
  4. Haz clic en "Execute"

✅ Código 204: Eliminada exitosamente. No retorna ningún contenido — eso es intencional, el estándar HTTP dice que 204 no tiene body.
❌ Código 404: Ya no existe o el ID es incorrecto.

BONUS
Prueba el validador — envía datos inválidos a propósito

Prueba que la validación funciona enviando datos incorrectos en el POST. Copia este payload en el "Request body":

JSON — payload inválido a propósito
{ "encuestado": { "nombre": "Ana", "edad": 999, "estrato": 9, "departamento": "Texas" }, "respuestas": [{ "pregunta_id": 1, "pregunta_texto": "Test", "respuesta": 10, "tipo": "likert" }] }

✅ Qué deberías ver: Código 422 con un JSON que lista exactamente cuántos errores hubo y en qué campo está cada uno. Así funciona la "aduana transaccional" del sistema.
